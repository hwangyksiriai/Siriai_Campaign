# PRD — SIRIAI 캠페인 허브 (Influencer Campaign Hub)

> 한 개의 링크에서 여러 캠페인을 한눈에 보고, 캠페인별 가이드를 확인한 뒤 바로 신청하는 인플루언서용 셀프서비스 허브.
> **별도 관리자(어드민) 페이지 없이, 구글시트 하나로 캠페인 등록과 신청 취합을 모두 처리한다.**

- **작성일**: 2026-06-21
- **담당**: 인플루언서 마케팅 (SIRIAI)
- **레퍼런스**: https://oddtype-inviteb.vercel.app/ (단일 캠페인 모집 + 4단계 신청 흐름)
- **상태**: Draft v0.2 — 구글시트 연동 방식 확정 반영

---

## 1. 배경 & 문제 정의

현재 인플루언서 섭외는 **캠페인이 생길 때마다 개별로 연락 → 가이드 전달 → 신청 취합**을 반복한다. 캠페인 수가 늘수록 이 과정이 병목이 된다.

**핵심 문제**
- 캠페인마다 매번 1:1로 연락하고 같은 안내를 반복한다.
- 가이드(필수 해시태그, 공동작업자 계정, 촬영 가이드 등)가 메시지/파일로 흩어져 전달 누락·오류가 잦다.
- 신청자 정보(배송지, 인스타 계정, 동의)가 채널마다 제각각으로 모여 취합이 번거롭다.

**해결 방향**
> "이 링크 하나 보고 원하는 캠페인 골라서 신청하세요" 가 되는 **상시 허브 1개**.
> 인플루언서는 목록에서 캠페인을 둘러보고, 상세에서 가이드까지 확인한 뒤, 그 자리에서 신청한다.
> 담당자는 **구글시트에 캠페인을 한 줄 적기만** 하면 사이트에 자동 노출되고, **신청은 같은 시트로 자동 취합**된다.

## 2. 목표 & 성공지표 (KPI)

| 목표 | 지표 |
|---|---|
| 섭외 운영공수 절감 | 캠페인당 담당자 연락/안내 메시지 수 ↓ |
| 신청 전환 | 링크 방문 → 신청 완료 전환율 |
| 가이드 전달 정확도 | 가이드 미준수로 인한 콘텐츠 반려율 ↓ |
| 모집 속도 | 캠페인 오픈 → 목표 신청자 도달 시간 |
| 재방문/충성 | 동일 인플루언서의 복수 캠페인 신청 비율 |

## 3. 사용자 (Personas)

**P1. 인플루언서 (메인 사용자, 비로그인)**
- 모바일에서 링크를 받아 들어온다. 로그인 없이 둘러보고 신청한다.
- 알고 싶은 것: 뭐 주는데(고료/제품), 뭘 해야 하는데(콘텐츠 형식·분량), 언제까지(일정), 뭐가 필수인데(해시태그/멘션/공동작업자), 어떻게 신청해(배송지 입력).

**P2. 마케팅 담당자 (운영자)**
- **구글시트에 캠페인을 입력**해 사이트에 노출하고, **구글시트로 들어온 신청자**를 확인한다.
- 별도 관리자 페이지를 따로 쓰지 않는다. (= 새 어드민 없음)

## 4. 핵심 사용자 흐름 (User Flows)

### 4.1 인플루언서 신청 흐름 (메인)
```
링크 진입 (로그인 없이 누구나 열람)
  → [캠페인 목록] 카드로 전체 캠페인 한눈에 (필터/정렬 없음)
  → 카드 선택
  → [캠페인 상세] 브랜드 소개 · 개요(형식/고료/업로드 기간)
                 · 콘텐츠 제작 가이드(필수/금지/추천 + 가이드 파일)
                 · 업로드 가이드(공동작업자 계정 복사, 필수 해시태그 복사, 캡션 멘션)
  → [신청] 배송지 주소 + 상세주소 + 요청사항 + 인스타 핸들 + 필수 동의 체크
  → [완료] 신청 접수 확인 + 정산/업로드 안내
  → (신청 내용이 구글시트 '신청' 탭에 자동 기록)
```
- 상태 배지: `지원 가능` / `협의` / `마감` (구글시트의 상태 값 기준).

### 4.2 담당자 운영 흐름 (구글시트만 사용)
```
[구글시트 '캠페인' 탭] 새 캠페인을 한 줄 입력 / 수정 / 마감 처리
        ↓ (사이트가 시트를 읽어 자동 반영)
[웹 허브] 캠페인 목록·상세에 자동 노출
        ↓
[인플루언서] 신청
        ↓ (자동)
[구글시트 '신청' 탭] 신청자 정보가 한 줄씩 자동 누적 → 여기서 확인/선정
```
- **별도 관리자(어드민) 웹페이지를 만들지 않는다.** 구글시트가 등록·취합의 단일 도구.

## 5. 기능 요구사항 (Functional Requirements)

### 5.1 캠페인 목록 (허브 홈) — `/`
- 로그인 없이 **링크를 받은 누구나 전체 캠페인을 열람**.
- 활성 캠페인을 카드 목록으로 표시. 카드 구성: 브랜드명·로고, 추천도(%), 대표 이미지, 제목, 고료(₩), 지원 인원 수, 상태 배지.
- **카테고리 필터 없음**, **정렬 옵션 없음** (시트에 적힌 순서대로 단순 노출).
- 마감/비노출 캠페인은 숨김(또는 하단 "마감" 표시).
- 빈 상태: "현재 모집 중인 캠페인이 없어요" 안내.

### 5.2 캠페인 상세 — `/campaigns/[slug]`
화면 섹션(첨부 화면 기준):
1. **헤더**: 제목, 대표 이미지, 고료(예: ₩50,000), 속성 태그(릴스 / 공동작업 필수 / 2차활용 필수 / 지원 N명).
2. **브랜드 소개**: 브랜드 설명 텍스트.
3. **캠페인 개요**: 콘텐츠 형식(릴스/피드), 고료, 업로드 기간(시작~종료).
4. **콘텐츠 제작 가이드**:
   - 가이드 파일 보기(외부 링크/PDF)
   - ✅ 필수 사항 (예: 전 색상 발색)
   - ⛔ 금지 사항 (예: 타 브랜드 언급 금지)
   - 💡 추천 사항 (예: 고화질 촬영)
5. **업로드 가이드**:
   - 공동작업자 계정 (예: `@oddtype_official`, `@oddtype_jp`) — **각 항목 복사 버튼**
   - 캡션 필수 사항(멘션 안내)
   - 필수 해시태그 (예: `#무신사 #닥시젤리틴트`) — **복사 버튼**
6. **하단 고정 CTA**: `신청하기` / 이미 신청 시 `신청 완료`(비활성).
- **brand-guide 스킬과 동일한 데이터 구조**를 사용해 가이드 일관성 유지.

### 5.3 신청 폼 — 상세 화면 내 바텀시트/모달 (또는 `/campaigns/[slug]/apply`)
입력 필드(첨부 화면 + 레퍼런스 기준):
- 배송지 주소(주소 검색, 예: Daum 우편번호 API) + 상세 주소
- 요청사항(선택)
- 인스타그램 핸들/프로필 링크
- (캠페인에 따라) 이름·이메일·휴대폰·연령대 등 기본 정보
- **고지 안내**: "배송지는 매 캠페인마다 새로 확인합니다. 정확히 입력해 주세요." / 업로드·정산 안내(마감까지 업로드 시 정산, 매월 5일 일괄)
- **필수 동의 체크**: "선정 후 미업로드 시 제공받은 제품의 원가를 청구받는 데 동의합니다." 등 (캠페인별 동의 항목 설정 가능)
- 제출 버튼은 필수값/동의 충족 전 비활성. 제출 후 **완료 화면**(접수 확인 + 다음 안내).
- 제출 시 **구글시트 '신청' 탭에 한 줄 자동 기록**.
- **중복 신청 방지**: (캠페인 × 이메일/휴대폰) 기준으로 안내(가능하면 차단).

### 5.4 (선택) 내 신청 확인
- MVP에서는 이 기기 기준 localStorage로 "신청함" 표시 정도. 별도 조회 페이지는 후순위.

> ❌ **어드민(관리자 웹페이지) 없음** — 캠페인 등록·신청자 확인은 모두 구글시트에서 처리.

## 6. 데이터 모델 — 구글시트 (Single Source of Truth)

> 구글시트 1개 문서에 탭 2개. 사이트는 ①을 **읽고**, 신청은 ②에 **쓴다**.

### ① `캠페인` 탭 (담당자가 직접 입력)
| 컬럼 | 예시 | 설명 |
|---|---|---|
| slug | `oddtype-jun-w2` | URL용 고유값(자동/수동) |
| title | `오드타입 6월 2주차` | 캠페인 제목 |
| brand_name | `무신사` | 브랜드명 |
| brand_logo_url | (이미지 링크) | 로고 |
| brand_intro | (설명 텍스트) | 브랜드 소개 |
| cover_image_url | (이미지 링크) | 대표 이미지 |
| recommend_score | `97` | 추천도(%) |
| content_type | `릴스` | 콘텐츠 형식 |
| reward_amount | `50000` | 고료 |
| upload_start / upload_end | `2026-06-11` / `2026-06-17` | 업로드 기간 |
| flags | `공동작업 필수, 2차활용 필수` | 속성 태그(쉼표) |
| guide_file_url | (링크) | 가이드 파일 |
| required_notes / forbidden_notes / recommended_notes | (텍스트) | 필수/금지/추천 |
| collab_accounts | `@oddtype_official, @oddtype_jp` | 공동작업자 계정(쉼표) |
| required_hashtags | `#무신사 #닥시젤리틴트` | 필수 해시태그 |
| caption_requirements | (텍스트) | 캡션 멘션 안내 |
| consent_items | (텍스트, 줄바꿈) | 필수 동의 항목 |
| status | `open` / `closed` | 노출 여부/상태 |

### ② `신청` 탭 (자동 누적)
| 컬럼 | 설명 |
|---|---|
| timestamp | 신청 시각(자동) |
| campaign_slug / campaign_title | 어떤 캠페인 |
| name, email, phone, age_range | 신청자 기본 정보 |
| instagram_handle | 인스타 핸들 |
| address, address_detail | 배송지 |
| request_note | 요청사항 |
| agreed | 동의 여부 |

## 8. 비기능 요구사항

- **모바일 우선**: 트래픽 대부분 인스타 인앱 브라우저/모바일. 단일 컬럼, 큰 터치 타깃, 하단 고정 CTA.
- **성능/신선도**: 사이트는 구글시트를 주기적으로(또는 요청 시) 읽어 반영. 약간의 캐시 지연 허용(예: 1~5분) — 너무 잦은 시트 호출 방지.
- **공유 최적화**: OG 태그(캠페인별 썸네일/제목) — 카톡/인스타 공유 시 미리보기.
- **접근성**: 복사 버튼 피드백("복사됨"), 폼 에러 명확화, 색 대비.
- **개인정보**: 주소·연락처 수집 → 동의/이용목적 고지, 보관기간 정책. 구글시트 접근 권한 관리. (개인정보 처리방침 페이지 필요.)
- **안티스팸**: 중복 신청 방지 + (필요시) 간단한 봇 방지.

## 9. 기술 아키텍처 (제안)

- **프레임워크**: Next.js (App Router) — Vercel 배포(레퍼런스도 Vercel).
- **데이터 연동(구글시트)**:
  - **읽기(캠페인)**: Google Sheets API(서비스 계정) 또는 시트 공개 후 읽기. 사이트가 '캠페인' 탭을 읽어 목록/상세 생성.
  - **쓰기(신청)**: 신청 제출 → 서버(Next.js API Route)가 Google Sheets API로 '신청' 탭에 행 추가. (대안: Google Apps Script 웹앱 엔드포인트)
- **주소 검색**: Daum 우편번호(카카오) API.
- **이미지/파일**: 시트에 이미지 URL을 넣거나, 필요 시 Vercel Blob 사용.
- **분석**: Vercel Analytics + 퍼널 이벤트(조회→상세→신청).

## 10. 범위 (Scope) & 로드맵

**MVP (1차)** — "보고 신청까지 + 구글시트 연동"
- 캠페인 목록(필터/정렬 없음) · 캠페인 상세(전체 가이드 + 복사 버튼) · 신청 폼(주소검색·동의) · 완료 화면
- 구글시트 '캠페인' 탭 읽어 노출 / '신청' 탭에 자동 기록
- 중복 신청 방지, 모바일 UI, OG 공유

**2차**
- 내 신청 확인, 마감 캠페인 섹션, 통계/퍼널 고도화
- 선정 안내(influencer-dm), 검수(content-review), 정산(settlement-helper) 워크플로 연결

**3차**
- 알림(카톡/이메일), 성과 리포트(campaign-report) 자동화, 추천 매칭

## 11. 확정 사항 / 남은 결정

**확정**
- ✅ 카테고리 필터 없음 / 정렬 없음
- ✅ 별도 어드민(관리자 웹페이지) 없음
- ✅ 캠페인 등록 + 신청 취합 모두 **구글시트**에서 처리
- ✅ 로그인 없이 링크 받은 누구나 열람

**남은 결정 / 준비물**
1. **연동할 구글시트**: 새로 만들지 / 기존 시트 사용할지. (사이트가 접근하려면 시트 공유 또는 서비스 계정 권한 필요)
2. **신청 쓰기 방식**: Google Sheets API(서비스 계정) vs Google Apps Script 웹앱 — 둘 다 가능, 권한 설정 난이도 차이.
3. **주소 검색**: Daum 우편번호 API 사용 여부(권장).
4. **개인정보 처리방침/사업자 고지** 문구 소스.
5. **브랜딩**: 허브 자체 브랜드(SIRIAI) 노출 vs 캠페인/브랜드 중심.

## 12. 레퍼런스 대비 차별점

| | 레퍼런스(oddtype-inviteb) | 본 제품(허브) |
|---|---|---|
| 캠페인 수 | 1개(Type A/B) | **다수, 상시** |
| 탐색 | 없음(단일 페이지) | **여러 캠페인 카드 목록** |
| 가이드 | 페이지 내 고정 | **캠페인별 구조화 가이드 + 복사 버튼** |
| 운영 | 폼 수집 | **구글시트 1개로 등록·취합 일원화** |
| 재사용 | 캠페인마다 새 페이지 | **링크 1개 영구 재사용** |
